# Support

Use GitHub issues for reproducible bugs, release-readiness problems, setup failures, and feature requests.

Before opening an issue:

- Search existing issues.
- Run `npm run release:check` when the problem is related to code, build, tests, or release hygiene.
- Run `npm run init:baseline -- --json` when the problem is setup or local environment readiness.
- Remove secrets from logs before sharing output.

For vulnerabilities or token exposure, do not open a public issue with exploit details. Use the private process in [SECURITY.md](SECURITY.md).
